하나의 서버에는 여러 개의 프로젝트가 돌아간다.
그렇기에 Modules로 들어가 최상위 프로젝트를 설정해놔야 한다.
localhost:9090/day01/...
까지 해야 day01안에 설정 되어 있는 것들을 할 수 있다.

서블릿은 html 틀을 만들어 데이터까지 넣어준 다음에 정적으로 만들어 응답을 해주는 거고
JSP는 데이터만 정적으로 만들어 응답을 해주면 그 데이터의 위치에 맞게 응답쪽에 있는 html에 위치 시키는 것

JSP(Java Server Page)
		HTML을 중심으로 자바와 같이 연동하여 사용하는 웹 언어이다.
		HTML 코드 안에 JAVA 코드를 작성할 수 있다.

		jsp 파일이 뷰단이 아니라 jsp로 인해 만들어진 html이 뷰단이다.
		
		jsp 파일이 번역이 되면 서블릿 코드로 바뀐다.
		그 서블릿 코드가 실행되면서 html이 나오게 된다.

		선배들이 번역을 해주는 기능을 만들어놨기 때문에 jsp를 이용해
		서블릿을 이용한다.

		jsp를 실행하면 서블릿 클래스가 만들어지고 이게 나중에 컴파일이 되면서
		동적인 것을 정적인 것으로 가공하고 응답해준다.

서블릿(Servlet)
		JAVA 코드로 HTML을 작성할 수 있도록 해주는 JAVA 프로그램이다.

		out.print("<html>");
		out.print("<head>");
		out.print("<body>");
		...

		localhost:9090/test -> web.xml에 매핑해놓은 서블릿 찾기
		서블릿은 클래스이기 때문에 서블릿이 실행
		-> TestServlet

		1. Thread에 의해서 서블릿에 있는 service() 메소드가 호출된다.
		2. 전송 방식을 파악해서 doGet() 또는 doPost() 메소드가 호출된다.
		3. 각 메소드 안에 작성해놓은 내용들이 실행된다.
		  각 메소드 내에서는 동적인 것들을 정적으로 바꿔서 보내준다.

		- 이후 소멸단계
		WAS는 Response 객체를 HttpResponse형태(정적)로 바꿔서 웹 서버에
		전달하고 생성된 Thread를 종료한다. 그리고 HttpServletRequest,
		HttpServletResponse 객체를 제거한다.

스크립트 태그
		HTML 코드에 자바 코드를 넣어 프로그램이 수행하는 기능을 구현할 수 있다.

		1. 선언문(declaration)		: <%! %>
				자바 변수나 메소드를 선언, 정의하는데 사용하는 태그

		2. 스크립틀릿(scriptlet)	: <% %>
				자바 변수 선언 및 자바 로직 코드를 작성하는데 사용하는 태그

		3. 표현문(expression)		: <%= %>
				변수, 계산식, 메소드 호출의 리턴값 등을 표현해주는 태그
				HTML 문서 안에 작성한 값이 그대로 표현된다. 타입은 문자열이다.
